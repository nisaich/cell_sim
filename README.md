# Biofilm Simulation

## Что это

Это C++-проект с простой двумерной симуляцией роста клеточной колонии на решётке.
В каждой ячейке поля могут храниться:

- клетка;
- количество пищи;
- концентрация антибиотика.

Симуляция визуализируется через `raylib`.

## Структура проекта

- `main.cpp` - создание поля, начальная постановка клеток и запуск визуализации.
- `Field.hpp` / `Field.cpp` - поле, ячейки, соседи, шаг симуляции.
- `Biomass.hpp` / `Biomass.cpp` - состояния клеток: активные, неактивные и мёртвые.
- `Food.hpp` / `Food.cpp` - хранение и расход пищи.
- `Antibiotic.hpp` / `Antibiotic.cpp` - хранение концентрации антибиотика.
- `visualization.hpp` - интерфейс визуализации.
- `visualization.cpp` - базовая визуализация.
- `visualization_modified.cpp` - альтернативная визуализация окна.

## Сборка

Для сборки нужен компилятор с поддержкой C++17 и установленная `raylib`.

Вариант со стандартной визуализацией:

```bash
g++ -std=c++17 Biomass.cpp Field.cpp Food.cpp Antibiotic.cpp main.cpp visualization.cpp -o simulation_last -lraylib
```

Вариант с модифицированной визуализацией:

```bash
g++ -std=c++17 Biomass.cpp Field.cpp Food.cpp Antibiotic.cpp main.cpp visualization_modified.cpp -o simulation_last -lraylib
```

## Запуск

После сборки:

```bash
./simulation_last
```

## Что сейчас реализовано

- прямоугольное поле фиксированного размера;
- хранение пищи и антибиотика в каждой ячейке;
- активные клетки;
- неактивные клетки;
- мёртвые клетки, которые исчезают через несколько шагов;
- старение клеток;
- потребление пищи из среды;
- простая диффузия пищи по полю;
- деление активной клетки в свободную соседнюю ячейку;
- переход активной клетки в неактивное состояние, если рядом нет места;
- возврат неактивной клетки в активное состояние при достаточном количестве пищи;
- визуализация состояния поля.

## Что важно понимать

Это пока не полноценная биологическая модель биоплёнки.
Сейчас проект ближе к рабочему прототипу симуляции, чем к завершённой модели.

На данный момент есть несколько принципиальных ограничений:

- логика антибиотика как фактора выживания клеток ещё не встроена в шаг симуляции;
- резистентность клеток пока почти не влияет на поведение;
- параметры роста, старения и питания выбраны эвристически;
- модель не претендует на биологическую достоверность;
- часть переходов между состояниями клеток всё ещё очень упрощена.

## Как сейчас устроен шаг симуляции

Один шаг поля выполняет примерно такую последовательность:

1. Диффузия пищи между соседними ячейками.
2. Сбор списка живых клеток.
3. Проверка смерти клетки.
4. Увеличение возраста выживших клеток.
5. Потребление пищи из текущей ячейки.
6. Расход внутренних ресурсов клетки.
7. Попытка размножения.
8. Постепенное удаление мёртвых клеток.

## Начальные условия

В текущем `main.cpp`:

- создаётся поле размером `80 x 60`;
- во все ячейки кладётся одинаковое количество пищи;
- внизу поля ставится одна стартовая активная клетка;
- затем запускается визуализация.

## Что стоит улучшить дальше

- встроить антибиотик в логику шага симуляции;
- сделать влияние резистентности реальным, а не только визуальным;
- пересмотреть модель голодания и внутренних запасов клетки;
- вынести параметры модели в константы или конфиг;
- добавить сценарии инициализации поля;
- добавить нормальное описание правил модели и её ограничений;
- добавить Makefile или CMake.
