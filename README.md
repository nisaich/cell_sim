# Biofilm Simulation

Прототип двумерной симуляции роста клеточной колонии на решётке.

Проект написан на C++ и визуализируется через `raylib`. В каждой ячейке поля хранятся:

- клетка или пустое место;
- запас пищи;
- концентрация антибиотика.

Симуляция показывает состояние поля в реальном времени и параллельно рисует графики основных метрик:

- число живых и мёртвых клеток;
- общий запас пищи;
- средняя концентрация пищи;
- максимальная высота колонии.

## Возможности

- прямоугольное двумерное поле;
- активные, неактивные и мёртвые клетки;
- старение клеток;
- потребление пищи из окружающей среды;
- диффузия пищи по полю;
- размножение активных клеток;
- переходы между активным и неактивным состоянием;
- постепенное удаление мёртвых клеток;
- визуализация состояния поля и графиков в одном окне;
- запись статистики симуляции в `simulation_stats.csv`.

## Структура проекта

### Точка входа

- `main.cpp`
  Создаёт поле, инициализирует среду, размещает стартовую колонию и запускает визуализацию.

### Конфигурация

- `SimulationConfig.hpp`
  Центральный файл с параметрами симуляции и визуализации.

  В нём сгруппированы:
  - размеры поля;
  - стартовые условия;
  - параметры клеток;
  - параметры графиков;
  - параметры окна и режимов визуализации.

  Если нужно менять поведение модели, начинать стоит именно с этого файла.

### Модель среды

- `Field.hpp` / `Field.cpp`
  Реализация поля и отдельных ячеек.

  Основные обязанности:
  - хранение сетки `Cell`;
  - доступ к соседям;
  - размещение клеток;
  - шаг симуляции `make_one_step()`;
  - диффузия пищи `diffuse_food()`;
  - удаление мёртвых клеток через несколько шагов.

- `Food.hpp` / `Food.cpp`
  Простой контейнер для количества пищи в ячейке.

- `Antibiotic.hpp` / `Antibiotic.cpp`
  Контейнер для концентрации антибиотика.
  
  На текущем этапе антибиотик хранится в модели, но почти не участвует в биологической логике.

### Модель клеток

- `Biomass.hpp` / `Biomass.cpp`
  Основная логика клеток.

  Классы:
  - `abstract_Biomass` — базовый класс клетки;
  - `active_Biomass` — активная клетка;
  - `nonactive_Biomass` — неактивная клетка;
  - `dead_Biomass` — мёртвая клетка.

  Здесь реализованы:
  - возраст клетки;
  - расход и потребление пищи;
  - размножение;
  - переходы состояний;
  - поведение после смерти.

### Графики и статистика

- `Graphs.hpp` / `Graphs.cpp`
  Сбор и визуализация статистики симуляции.

  Что здесь есть:
  - `SimulationStats` — структура с метриками одного тика;
  - `collectStats(...)` — сбор статистики с поля;
  - `CsvStatsRecorder` — запись статистики в CSV;
  - `StatsHistory` — хранение истории и отрисовка графиков через `raylib`.

### Визуализация

- `visualization.hpp`
  Интерфейс визуализации.

- `visualization.cpp`
  Основной вариант визуализации.

- `visualization_modified.cpp`
  Альтернативный вариант визуализации окна и масштабирования.

Оба файла визуализации:

- вызывают шаг симуляции;
- рисуют поле;
- рисуют боковую панель графиков;
- используют одинаковую модель, но немного отличаются поведением окна.

## Как устроен шаг симуляции

Один шаг поля в `Field::make_one_step()` выполняет примерно такую последовательность:

1. Диффузия пищи между соседними ячейками.
2. Сбор списка живых клеток для текущего тика.
3. Проверка, должна ли клетка умереть.
4. Увеличение возраста выживших клеток.
5. Потребление пищи из среды.
6. Внутренний расход ресурсов клетки.
7. Попытка размножения или смены состояния.
8. Обработка мёртвых клеток и их удаление с задержкой.

## Как сейчас реализованы состояния клеток

### Active

Активная клетка:

- стареет;
- потребляет пищу;
- может размножаться в свободную соседнюю ячейку;
- при некоторых условиях может перейти в неактивное состояние.

### Nonactive

Неактивная клетка:

- живёт дольше активной;
- расходует меньше ресурсов;
- имеет изменённые коэффициенты устойчивости и потребления;
- может вернуться в активное состояние при выполнении условий, заданных в логике `Biomass.cpp`.

### Dead

Мёртвая клетка:

- не участвует в размножении;
- хранится на поле ограниченное число шагов;
- затем удаляется из ячейки.

## Конфигурация

Основные параметры меняются в `SimulationConfig.hpp`.

Главные секции:

- `simulation_config::field`
  Размер поля, стартовый уровень пищи, коэффициент диффузии.

- `simulation_config::colony`
  Размер и положение стартовой колонии.

- `simulation_config::biomass`
  Возраст клеток, расход пищи, шанс размножения, параметры переходов состояний.

- `simulation_config::graphs`
  Размеры и стиль графиков.

- `simulation_config::visualization`
  Размеры окна, отступы, нормализация цвета и режим отображения.

## Сборка

Требования:

- компилятор с поддержкой C++17;
- установленная `raylib`.

### Сборка с основной визуализацией

```bash
g++ -std=c++17 Biomass.cpp Field.cpp Food.cpp Antibiotic.cpp Graphs.cpp main.cpp visualization.cpp -o simulation_last -lraylib
```

### Сборка с альтернативной визуализацией

```bash
g++ -std=c++17 Biomass.cpp Field.cpp Food.cpp Antibiotic.cpp Graphs.cpp main.cpp visualization_modified.cpp -o simulation_test_mod -lraylib
```

## Запуск

После сборки:

```bash
./simulation_last
```

или:

```bash
./simulation_test_mod
```

## Выходные данные

Во время работы симуляции создаётся файл:

- `simulation_stats.csv`

В нём по каждому тику сохраняются:

- `tick`
- `liveCells`
- `deadCells`
- `emptyCells`
- `avgFood`
- `totalFood`
- `maxHeight`

## Что важно понимать

Это не законченная биологическая модель, а рабочий исследовательский прототип.

На текущем этапе:

- антибиотик уже есть в структуре данных, но почти не встроен в поведение клеток;
- многие коэффициенты выбраны эвристически;
- часть логики переходов между состояниями находится в активной доработке;
- модель ориентирована скорее на экспериментирование с правилами, чем на биологическую точность.

## Что можно улучшить дальше

- встроить антибиотик в правила выживания и переходов;
- сделать параметры более физически или биологически обоснованными;
- добавить сценарии инициализации поля;
- разделить визуализацию и модель ещё чище;
- добавить `Makefile` или `CMake`;
- покрыть ключевую логику тестами;
- стабилизировать правила переходов между активным и неактивным состоянием.
