# README

## Компиляция

Для сборки проекта нужен `raylib`, а также установленный заголовочный файл `raylib.h`.

Вариант сборки с модифицированной визуализацией:

```bash
g++ main.cpp Cell.cpp Field.cpp Food.cpp Antibiotic.cpp visualization_modified.cpp -o simulation_last -lraylib
```

Или вариант со стандартной визуализацией:

```bash
g++ main.cpp Cell.cpp Field.cpp Food.cpp Antibiotic.cpp visualization.cpp -o simulation_last -lraylib
```

Если `raylib.h` не установлен вручную в системе, проект не скомпилируется.

## Что это за проект

Это симуляция клеток на двумерном поле. У каждой ячейки поля (`Nucleus`) могут быть:

- клетка;
- запас пищи;
- концентрация антибиотика.

Поле хранится в классе `Field`, клетки описаны через иерархию:

- `abstract_Cell` - базовый класс;
- `active_Cell` - активная клетка;
- `nonactive_Cell` - неактивная клетка;
- `dead_Cell` - мёртвая клетка, которая затем исчезает.

Визуализация сделана через `raylib` и умеет отображать поле с разными режимами окраски.

## Что уже реализовано

- Проект разнесён по отдельным файлам: логика клеток, поля, пищи, антибиотика и визуализации.
- Есть класс поля `Field` и класс ячейки `Nucleus`.
- Поле поддерживает цилиндрическую топологию по оси `x`: выход за левую/правую границу нормализуется.
- Есть поиск соседей и свободных соседних ячеек.
- Есть размещение клеток на поле.
- Реализованы активные, неактивные и мёртвые клетки.
- У клетки есть возраст, внутренний запас пищи, максимальный возраст и уровень резистентности.
- Реализована базовая смерть клетки по возрасту или по исчерпанию внутренней пищи.
- После смерти клетка превращается в `dead_Cell` и спустя несколько шагов удаляется с поля.
- Активная клетка умеет делиться в свободную соседнюю ячейку.
- Если активной клетке некуда делиться, она может перейти в неактивное состояние.
- Неактивная клетка тратит меньше внутренней пищи и при подходящих условиях может снова стать активной.
- В `Food` уже есть базовый контейнер для количества пищи и метод `take(...)`.
- В `Antibiotic` уже есть базовый контейнер для концентрации антибиотика и методы изменения концентрации.
- Есть визуализация через `raylib`.
- Есть два варианта визуализации:
  - `visualization.cpp` - полноэкранный режим;
  - `visualization_modified.cpp` - окно подгоняется под размер поля и экрана аккуратнее.
- Поддержаны режимы окраски клеток по возрасту, резистентности и уровню питания среды.

## Что сейчас работает не до конца

### 1. Логика пищи реализована только частично

Сейчас пища формально присутствует в каждой ячейке поля, и клетка умеет вызывать `food_consumption_from_environment(...)`, но модель питания пока неполная:

- в `Nucleus` по умолчанию создаётся бесконечный запас пищи (`start_food = 1.0f / 0.0f`);
- из-за этого клетки фактически живут в среде с бесконечным ресурсом;
- нет нормального истощения среды;
- нет пополнения пищи по законам модели;
- нет зависимости поведения клеток от реального локального дефицита пищи;
- нет сценария, где распределение пищи по полю влияет на динамику колонии.

Итог: заготовка под еду есть, но нормальной работы пищи как механики симуляции пока нет.

### 2. Антибиотик пока не реализован как часть симуляции

Сейчас антибиотик существует только как объект данных в `Nucleus`:

- есть класс `Antibiotic`;
- есть хранение концентрации в ячейке;
- есть методы `add(...)` и `decrease(...)`;
- есть чтение значения для визуализации.

Но в логике шага симуляции антибиотик вообще не участвует:

- клетки не получают урон от антибиотика;
- резистентность клеток ни на что не влияет;
- антибиотик не меняет вероятность смерти, деления или перехода в неактивное состояние;
- нет распространения антибиотика по полю;
- нет внесения антибиотика в среду из `main.cpp` или из логики `Field`;
- в визуализации антибиотик пока не используется как отдельный режим отображения.

Итог: работа антибиотика сейчас отсутствует практически полностью.

## Текущий порядок шага симуляции

Сейчас `Field::make_one_step()` делает следующее:

1. Собирает список живых клеток.
2. Проверяет смерть по возрасту и внутренней пище.
3. Если клетка не умерла, увеличивает возраст.
4. Пытается забрать пищу из текущей ячейки.
5. Уменьшает внутренний запас пищи на шаг.
6. Пытается размножить клетку.
7. Обрабатывает постепенное исчезновение мёртвых клеток.

Важно: на этом шаге пока нет никакой реальной обработки антибиотика.

## Основные файлы

- `main.cpp` - запуск симуляции и начальная постановка клеток.
- `Cell.hpp` / `Cell.cpp` - логика клеток и их состояний.
- `Field.hpp` / `Field.cpp` - поле, ячейки, соседи, шаг симуляции.
- `Food.hpp` / `Food.cpp` - контейнер и операции для пищи.
- `Antibiotic.hpp` / `Antibiotic.cpp` - контейнер и операции для антибиотика.
- `visualization.cpp` - стандартная визуализация.
- `visualization_modified.cpp` - модифицированная визуализация окна.
