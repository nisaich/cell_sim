# Кратко о проекте

Этот проект моделирует рост колонии клеток на двумерном поле.

Поле состоит из ячеек `Nucleus`, и каждая ячейка может хранить:

- одну клетку;
- запас еды;
- значение антибиотика.

Старт симуляции задаётся в `main.cpp`: живая колония размещается в нижней строке поля, после чего программа шаг за шагом обновляет состояние и выводит его в консоль.

## Как работает симуляция

На каждом шаге:

1. проверяется, не должна ли клетка умереть;
2. выжившие клетки стареют;
3. клетки забирают еду из своей ячейки;
4. клетки тратят часть внутреннего запаса еды;
5. при достаточном запасе еды клетки размножаются;
6. мёртвые клетки удаляются с поля.

Клетка умирает, если:

- достигла максимального возраста;
- у неё закончилась внутренняя еда.

## Как происходит размножение

Размножение реализовано у `active_Cell`.

Правила такие:

- клетка должна иметь достаточно еды;
- у неё должна быть хотя бы одна свободная соседняя ячейка;
- новая клетка ставится в одну случайно выбранную свободную соседнюю клетку;
- дочерняя клетка получает половину еды родителя.

Соседи считаются только по 4 направлениям:

- вверх;
- вниз;
- влево;
- вправо.

Поэтому деление не идёт по диагонали.

Важно:

- выбор направления локально случайный;
- но старт колонии происходит снизу;
- поэтому в начале рост идёт в основном вверх и в стороны, потому что вниз от нижней границы расти нельзя.

## Особенности поля

- по горизонтали поле зациклено;
- по вертикали границы обычные;
- еда в ячейках конечна и не восстанавливается.

Это значит, что колония сначала может расширяться, но со временем из-за исчерпания еды и старения клеток вымирает.

## Что находится в файлах

`main.cpp`

- запуск программы;
- создание поля;
- начальное размещение клеток;
- основной цикл симуляции.

`Field.hpp` / `Field.cpp`

- описание поля;
- хранение ячеек;
- поиск соседей;
- выполнение одного шага симуляции.

`Cell.hpp` / `Cell.cpp`

- базовый класс клетки;
- живая клетка `active_Cell`;
- мёртвая клетка `dead_Cell`;
- заготовка `nonactive_Cell`.

`Food.hpp` / `Food.cpp`

- логика хранения и выдачи еды.

`Antibiotic.hpp` / `Antibiotic.cpp`

- хранение антибиотика в ячейке;
- в текущей версии почти не используется.

`visualization.hpp` / `visualization.cpp`

- консольная отрисовка поля.

## Итог

Это базовая модель клеточного роста на сетке, где клетки:

- едят;
- стареют;
- делятся;
- умирают;
- временно остаются на поле после смерти.

Полная подробная документация лежит в файле `README.md`.
